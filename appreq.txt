# PartsWale Rider Web App — Full Requirements Document
**Hosted at:** `rider.partswale.in`
**Type:** Mobile-first web app (HTML/CSS/JS or React)
**Auth:** OTP via WhatsApp → session token stored in sessionStorage
**Backend:** Supabase (direct REST calls or JS SDK) + n8n webhooks
**Target device:** Android phones, Chrome browser

---

## TECH STACK DECISIONS

- **Frontend:** React (single page app) hosted on Netlify
- **Auth storage:** `sessionStorage` (not localStorage — tab-scoped, cleared on close)
- **Supabase access:** Via Supabase JS SDK with Row Level Security (RLS) enabled
- **GPS pings:** Handled by GPSLogger app separately — NOT by this web app
- **Photo capture:** Built into this web app using `getUserMedia` camera API
- **Geo-fence check:** Done server-side in n8n webhook, not client-side
- **Real-time updates:** Supabase Realtime subscriptions for order status changes

## AUTHENTICATION SYSTEM

### How it works
1. Rider enters phone number on login page
2. Web app calls n8n webhook `POST /rider-send-otp`
3. n8n generates 6-digit OTP, stores in `riders.otp` and `riders.otp_expires_at` (now + 5 min), sends via WhatsApp text message
4. Rider enters OTP on web app
5. Web app calls n8n webhook `POST /rider-verify-otp`
6. n8n checks OTP matches and is not expired, generates a UUID session token, stores in `riders.session_token`, returns it
7. Web app saves token to `sessionStorage` as `rider_token` and rider's `id` as `rider_id`
8. All subsequent API calls include `rider_token` in headers
9. n8n validates token by querying `riders` where `session_token = token`
10. If rider not found in `riders` table → show "Not registered" error, do not allow login

### Session expiry
- Session token is valid until rider logs out or a new login overwrites it
- If `sessionStorage` is cleared (tab closed), rider must log in again
- No persistent login — intentional for security

**Screen 2 — OTP Entry (shown after OTP sent)**
- "OTP aapke WhatsApp par bheja gaya hai"
- 6-digit OTP input (individual boxes, auto-advance)
- "Verify Karo" button
- "OTP nahi mila? Dobara bhejo" link (cooldown: 30 seconds)
- Error state: "OTP galat hai ya expire ho gaya"
- Success: redirect to `/dashboard`

### API Calls

`POST https://n8n.srv1469471.hstgr.cloud/webhook/rider-send-otp`
```json
{ "phone": "917209463686" }
```
n8n flow: Query `riders` where `phone = phone`. If not found → return `{ success: false, reason: "not_registered" }`. If found → generate OTP, update `riders.otp` and `riders.otp_expires_at`, send WhatsApp message, return `{ success: true }`.

`POST https://n8n.srv1469471.hstgr.cloud/webhook/rider-verify-otp`
```json
{ "phone": "917209463686", "otp": "123456" }
```
n8n flow: Query `riders` where `phone = phone`. Check `otp matches` AND `otp_expires_at > now()`. If valid → generate UUID token, set `riders.session_token = token`, clear `riders.otp`, return `{ success: true, token: "uuid", rider_id: "uuid", name: "Suyash" }`. If invalid → return `{ success: false, reason: "invalid_otp" }`.

### sessionStorage after login
```
rider_token = "uuid-token"
rider_id    = "uuid-rider-id"
rider_name  = "Suyash"
```

### STEP 2 — Arrive at Pickup (Geo-fence Check)
**Condition to show:** `order.status = rider_assigned`, rider must be physically within 50m

**UI:**

- Web app reads current GPS
- Calls `POST /rider-at-pickup` with `{ rider_id, order_id, lat, lng }`
- n8n validates: distance from rider coords to `orders.dealer_lat/dealer_lng` ≤ 50m

**Supabase Realtime:** Subscribe to `orders` where `id = order_id`, watch for `dealer_confirmed_handoff` to flip to `true`

## PAGE 5 — PHOTO CAPTURE (`/capture/:order_id/:type`)

### Purpose
Full-screen camera tool for taking verified pickup/delivery photos. This is the most security-critical page.

### Flow

**Step 1 — Camera Opens**
- Full screen camera viewfinder using `getUserMedia({ video: { facingMode: 'environment' } })` (rear camera)
- Overlay frame guide: dashed rectangle with text "Saare items frame mein rakhein"
- Bottom: "Photo Lo" shutter button (large, circular)
- Top left: Back button (with warning: "Photo lena zaroori hai")

**Step 2 — Photo Taken**
- Viewfinder freezes, shows captured image
- App immediately calls `navigator.geolocation.getCurrentPosition()` (high accuracy)
- Loading indicator: "Location le rahe hain..."
- If GPS not available within 10 seconds → show error: "GPS signal nahi mila. Bahar jaao aur try karo"

**Step 3 — Preview + Metadata**
- Photo preview shown full screen
- Overlay badge (bottom of image):
  - 📍 Lat, Lng (rounded to 4 decimal places)
  - 🕐 Timestamp (IST)
  - 📦 Order ID (first 8 chars)
- "Retake" button (left) and "Submit" button (right, green)

**Step 4 — Submission**
On "Submit":
1. Convert image to base64 or blob
2. Upload to Supabase Storage bucket `order-photos` at path `/{order_id}/{type}_{timestamp}.jpg`
3. Get public URL from Supabase Storage
4. Call `POST /rider-confirm-photo`:
```json
{
  "rider_id": "uuid",
  "order_id": "uuid",
  "type": "pickup",
  "image_url": "https://supabase.../order-photos/...",
  "lat": 25.6234,
  "lng": 87.4823,
  "captured_at": "2026-04-10T15:30:00+05:30"
}
```
5. n8n validates:
   - GPS within 50m of expected location (dealer for pickup, mechanic for delivery)
   - `captured_at` within last 3 minutes
   - `dealer_confirmed_handoff = true` (for pickup only)
6. If valid:
   - Insert row in `photos` table
   - Update `orders.pickup_photo_id` or `orders.delivery_photo_id`
   - Update `orders.status`
   - Return `{ success: true }`
7. If invalid:
   - Return `{ success: false, reason: "out_of_geofence" / "photo_too_old" }`
   - Show error on screen: "Photo valid nahi hai — aap sahi jagah par nahi hain"
   - Allow retake

**Loading state during submission:**
- Full screen overlay: "Photo verify ho rahi hai..."
- Cannot go back during this

**On success:**
- Show "Photo accepted ✅" for 2 seconds
- Navigate back to `/order/:order_id`
- Order detail page refreshes and advances to next step

### Supabase Storage Setup Required
- Bucket name: `order-photos`
- Access: Private (only accessible via service key from n8n, or signed URLs)
- Path format: `/{order_id}/{type}_{unix_timestamp}.jpg`

### Supabase Query
```
SELECT orders.id, orders.delivery_confirmed_at, orders.rider_paid_at,
       quotes.quote_details, users_mechanic.district as mechanic_district,
       users_dealer.shop_name as dealer_name
FROM orders
JOIN quotes ON orders.quote_id = quotes.id
JOIN users users_mechanic ON orders.mechanic_id = users_mechanic.id
JOIN users users_dealer ON orders.dealer_id = users_dealer.id
WHERE orders.rider_id = '{rider_id}'
AND orders.status IN ('completed', 'delivered')
ORDER BY orders.delivery_confirmed_at DESC
LIMIT 20
```

## SECURITY RULES

- All webhooks except `/rider-send-otp` and `/rider-verify-otp` require `rider_token` in header: `Authorization: Bearer {token}`
- n8n validates token by querying `riders` where `session_token = token` before processing any request
- Photo uploads go directly to Supabase Storage using a short-lived signed upload URL (generated by n8n, passed to frontend) — frontend never holds the service key
- Geo-fence validation is ALWAYS done server-side in n8n — frontend GPS is used only for UI feedback (distance indicator), never trusted for actual validation
- OTP expires in 5 minutes
- Session token is invalidated on logout

## SUPABASE REALTIME SUBSCRIPTIONS

The web app subscribes to these channels after login:

```javascript
// Watch for order status changes
supabase
  .channel('order-updates')
  .on('postgres_changes', {
    event: 'UPDATE',
    schema: 'public',
    table: 'orders',
    filter: `rider_id=eq.${rider_id}`
  }, (payload) => {
    // Update UI based on new status
    handleOrderStatusChange(payload.new)
  })
  .subscribe()

// Watch for dealer handoff confirmation
supabase
  .channel('handoff-updates')
  .on('postgres_changes', {
    event: 'UPDATE',
    schema: 'public',
    table: 'orders',
    filter: `id=eq.${active_order_id}`
  }, (payload) => {
    if (payload.new.dealer_confirmed_handoff) {
      unlockPhotoStep()
    }
  })
  .subscribe()
```

- Photo upload: < 10 seconds (compress image to max 1MB before upload)
- All API calls timeout after 15 seconds with error shown

## IMAGE COMPRESSION BEFORE UPLOAD

Before sending photo to Supabase Storage:
1. Draw captured frame to an offscreen `<canvas>`
2. Resize to max 1280px width (maintain aspect ratio)
3. Export as JPEG at 80% quality
4. Result: typically 200-500KB per photo
5. Upload compressed blob
